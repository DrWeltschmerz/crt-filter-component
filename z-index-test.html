Z-INDEX STACKING TEST

This page demonstrates how the CRT overlay interacts with elements at different z-index layers. Use the controls to adjust the overlay's z-index and see which elements appear above or below it.

How to test:
1. Open CRT controls (bottom-right)
2. Change the "Mode" dropdown
3. Watch how the overlay interacts with buttons and sections
Modes:
- Disabled: Overlay is hidden
- Behind Content: Overlay at z-index -1, below page elements
- On Top: Overlay at z-index 9999, above everything

Section at z-index: -1

This section should be behind most content but above the background image (z:-2). When overlay is at z:-1, they share the same layer (DOM order determines visibility).

Section at z-index: 8

Expected Behavior

  • Mode: Disabled - Overlay is completely hidden, no visual effect
  • Mode: Behind Content - Overlay at z-index: -1, below all page elements, effects on background only
  • Mode: On Top - Overlay at z-index: 9999, above all page elements including buttons and content sections

Note: The overlay now uses a simple 3-mode system for reliable z-index positioning. For z-index to work properly, elements must have position: relative, absolute, or fixed. Static elements ignore z-index.
